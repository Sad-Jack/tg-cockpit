# /update-memory — обслуживание памяти сущности NAME

Цель: держать `channels/NAME/CLAUDE.md` и `channels/NAME/skills/voice.md` компактными и актуальными. Это дешёвый по токенам контекст, который читается перед каждой задачей — не раздувать. Сущность NAME — это канал ИЛИ группа.

## Шаги

1. **Прочитать память.** Открой оба файла:
   - `channels/NAME/CLAUDE.md` — рабочая память «мозга»: Snapshot, Аудитория, Голос и тон, Контент-столпы, Выученные уроки, Расписание, Do/Don't.
   - `channels/NAME/skills/voice.md` — профиль голоса: Тема, Аудитория, Голос и тон, Длина, Эмодзи, Форматирование, Правила стиля.

2. **Консолидировать и чистить устаревшее.** Пройди по «Выученным урокам» и профилю голоса:
   - Объедини дубли и близкие по смыслу пункты в один.
   - Удали устаревшие и противоречивые: если новый урок отменяет старый — оставь только новый.
   - Датированные уроки чисти — старые убирай, оставляй свежие и всё ещё применимые. Цель — короткий список действующих правил, а не журнал.
   - Если устойчивый урок про голос/тон осел в CLAUDE.md — перенеси его в `voice.md` (там ему место) и убери дубль из CLAUDE.md.

3. **Освежить Snapshot.** Возьми самый поздний файл из `channels/NAME/insights/<date>.md`. Перепиши Snapshot в CLAUDE.md 2–4 строками: текущее состояние, ключевые цифры/выводы. Не копируй insights целиком — только выжимку.

4. **Свериться с фактами.** При расхождении сверь `config.yaml` (handle, kind, столпы, расписание, tz, exclusions) и подправь рассинхрон в соответствующих секциях обоих файлов.

5. **Держать компактно.** Редактируй существующие секции на месте через Edit, не добавляй новые заголовки. Длинные перечни сворачивай, воду режь. Чем дешевле контекст — тем чаще и точнее он отрабатывает.

## Правила
- Только осмысленные правки секций; новые заголовки не плодить, структуру не ломать.
- Голос/тон/стиль живут в `voice.md`; оперативная память и уроки — в `CLAUDE.md`.
- Не трогать данные в `data/` (gitignored) и черновики в `drafts/`.
- Сеть не дёргаем — это чисто работа с файлами памяти.
