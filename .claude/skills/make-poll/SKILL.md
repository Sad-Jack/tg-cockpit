# Создание опроса / квиза

Опросы — сильный формат вовлечения: ими удобно проверять гипотезы (см. идеи из `/analyze-channel`), греть аудиторию и собирать обратную связь без написания поста.

## Шаги

0. Гард изучения (см. CLAUDE.md): post/schedule/poll create/group reply требуют studied: true; если false — предложи /study-entity и остановись.

1. **Сформулируй вопрос и варианты.** Возьми тему и тон из `channels/NAME/skills/voice.md`. Если voice.md пустой/скаффолд — предложи /study-entity или скажи, что стиль ещё не выведен (см. CLAUDE.md). Вариантов должно быть от 2 до 10. Вопрос и варианты — короткий текст; если нужна разметка — /format-tg; без таблиц.

2. **Обычный опрос:**
   ```
   uv run tgcockpit poll create --channel NAME --question "Q" --option "A" --option "B" --option "C"
   ```
   Добавь `--multiple`, если допустим выбор нескольких вариантов.
   Команда встанет в очередь на кнопку — скажи пользователю, что готово и ждёт подтверждения; не утверждай, что опубликовано (см. CLAUDE.md).

3. **Квиз (один правильный ответ):** к команде выше добавь `--quiz --correct N`, где `N` — индекс правильного варианта, нумерация с 0 (первый `--option` = 0):
   ```
   uv run tgcockpit poll create --channel NAME --question "Q" --option "A" --option "B" --quiz --correct 1
   ```

## Примеры вопросов в голосе сущности

Эталон стиля — `channels/NAME/skills/voice.md` (на «ты», честно, по теме столпов). Например, для канала про вход в ML/AI:

- «Что путало тебя дольше всего на старте в ML?» — варианты: «Precision vs recall», «Bias vs variance», «Fine-tuning vs RAG», «Всё сразу 🙈».
- «Honestly, как у тебя с пет-проектами?» — варианты: «Несколько уже сделал», «Один в работе», «Только собираюсь», «Стыдно, ноль».
- «Какой разбор зайдёт следующим?» — варианты: «RAG на пальцах», «Метрики без боли», «n8n для ML».

## Замечания

- Опросы работают одинаково в каналах и группах (см. CLAUDE.md).
- `poll create` отправляется сразу, без `--at` (расписания для опросов нет).
- Вариантов — от 2 до 10.
- Опрос нельзя отредактировать текстом после публикации — чтобы изменить вопрос или варианты, удали старый (`message delete --channel NAME --id MSGID`) и создай заново.
