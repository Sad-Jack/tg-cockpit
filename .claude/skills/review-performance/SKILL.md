# /review-performance

Цель: замкнуть петлю обратной связи. Сверить ФАКТ (реальные метрики опубликованных постов) с ПРОГНОЗОМ/ГИПОТЕЗАМИ (ожидания из `content-plan.md` и гипотезы из `CLAUDE.md`), дописать датированные «Выученные уроки» и при необходимости подправить профиль голоса.

`NAME` — имя сущности (канал ИЛИ группа). Подставь его во всех командах и путях. Источник данных после фетча: `channels/NAME/data/history.json`.

Перед стартом: Гард изучения (см. CLAUDE.md): post/schedule/poll create/group reply требуют studied: true; если false — предложи /study-entity и остановись. `fetch-history --full` опирается на изученную сущность — без `study` сверять нечего.

## Чек-лист действий

### 1. Освежить метрики недавно опубликованных постов
```
uv run tgcockpit fetch-history --channel NAME --full
```
- `--full` подтягивает свежие метрики (просмотры/реакции/пересылки) по уже опубликованным постам, а не только новые сообщения.
- Источник данных после этого: `channels/NAME/data/history.json`.

### 2. Пересчитать аналитику
```
uv run tgcockpit analytics --channel NAME
```
- Сохраняет аналитику в `channels/NAME/insights/<date>.json` и отчёт в `channels/NAME/insights/<date>.md`.
- Зафиксируй ключевые цифры: лучшие/худшие посты, охват по столпам (pillars), время/слоты, динамика к прошлым периодам.

### 3. Сверить факт с прогнозом и гипотезами
- Гипотезы: открой `channels/NAME/CLAUDE.md` — какие гипотезы о голосе/темах/времени там зафиксированы; подтвердились или опровергнуты данными.
- Ожидания: открой `channels/NAME/content-plan.md` — найди ожидания по недавним постам (какой столп, какой слот, какой ожидался отклик).
- Сопоставь каждый недавний пост: где факт превзошёл прогноз, где недотянул. При необходимости сверься с черновиками `channels/NAME/drafts/*.md` (frontmatter: `status: posted`, `pillar`, `title`, `scheduled_msg_id`).
- На каждый пост заполни сравнительную строку: `[Дата/Заголовок] · [Ожидание плана] · [Факт] · [Дельта] · [Объяснение]`.
- Сформулируй 2–4 конкретных вывода: что сработало / что не сработало и почему (тема, столп, формат, время, длина, медиа).
- Жизненный цикл гипотез: опровергнутую гипотезу помечай `[REJECTED YYYY-MM-DD]` прямо в `channels/NAME/CLAUDE.md`, чтобы /content-plan её больше не тащил.

### 4. Дописать «Выученные уроки» в память сущности
- Файл: `channels/NAME/CLAUDE.md`, секция **Выученные уроки** (датированные записи).
- Порог записи: фиксируй урок, только если дельта метрики >10% от прогноза ИЛИ урок опровергает гипотезу. Мелкий шум в уроки не пишем.
- Добавь новую датированную запись (формат даты `YYYY-MM-DD`), НЕ перезаписывая старые. Каждый урок — одна строка: что наблюдали (факт vs прогноз/гипотеза) → какой вывод/правило на будущее.
- Пример строки: `- 2026-06-07: посты по столпу «X» в утренний слот дают +N% охвата против прогноза → ставить «X» на утро.`
- Уроки в `CLAUDE.md` лонгитюдны — при сверке НЕ удаляй старые записи (чистка/консолидация — задача /update-memory). Дисциплина памяти — см. /update-memory.
- Если вывод меняет правила — обнови также секции **Расписание постинга** и/или **Do/Don't** в том же `CLAUDE.md`.

### 5. Подправить профиль голоса (если стиль эволюционировал)
- Файл: `channels/NAME/skills/voice.md`.
- Триггер правки: данные показывают, что тон/длина/структура сдвинулись (зашёл другой тон, длина, структура, эмодзи, формулы CTA, формат заголовков) — точечно дополни/перепиши конкретные пункты профиля, чтобы будущие черновики попадали в актуальный голос.
- Если voice.md пустой/скаффолд — предложи /study-entity или скажи, что стиль ещё не выведен (см. CLAUDE.md).

## Готово, когда
- `fetch-history --full` и `analytics` отработали без ошибок, метрики свежие.
- Каждый недавний пост сопоставлен с прогнозом из `content-plan.md` и гипотезами из `CLAUDE.md`.
- В `channels/NAME/CLAUDE.md` появилась новая датированная запись в «Выученные уроки».
- При необходимости обновлён `channels/NAME/skills/voice.md`.
