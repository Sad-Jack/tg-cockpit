# Форматирование для Telegram

Канонические детали форматирования. Жёсткое правило — в `CLAUDE.md`; здесь — шпаргалка и крайние случаи.

## Главное различие: пост vs ответ в чат

- **Тело поста, `message edit`** → публикуется через Telethon с `parse_mode=html` **как есть, без конвертации**. Пиши **HTML руками**. Markdown-звёздочки (`**`, `_`) в посте не отрендерятся — подписчик увидит сырые символы.
- **Ответ пользователю в чат** → мост сам прогоняет твой текст через конвертер (`md_to_telegram_html`). Тут можно писать обычным **markdown**: `**жирный**`, `` `код` ``, ```` ```блоки``` ````, `# заголовок` (→ жирный), `[текст](https://…)`, и даже markdown-таблицы (станут списками). Подчёркивания `_` курсивом НЕ становятся — поэтому `ML_Road` и `snake_case` не ломаются.

То есть HTML руками нужен именно для постов. В чат — пиши естественно.

## Шпаргалка HTML-тегов (для постов)

- Жирный — `<b>текст</b>`
- Курсив — `<i>текст</i>`
- Подчёркнутый — `<u>текст</u>`
- Зачёркнутый — `<s>текст</s>`
- Моноширинный инлайн — `<code>текст</code>`
- Блок кода — `<pre>текст</pre>` или `<pre><code class="language-python">…</code></pre>`
- Ссылка — `<a href="https://example.com">текст</a>`
- Цитата — `<blockquote>текст</blockquote>`
- Спойлер — `<tg-spoiler>текст</tg-spoiler>`

Теги вкладываются: `<b><i>жирный курсив</i></b>`. Это полный список тегов, которые понимает Telegram, — других не используй.

## Экранирование — ТОЛЬКО три символа

В обычном тексте поста экранируй `<` → `&lt;`, `>` → `&gt;`, `&` → `&amp;`. Больше ничего: кавычки, точки, эмодзи, скобки — как есть.

Крайние случаи:
- Внутри `<a href="URL">` сам URL не экранируй повторно — `&` в параметрах ссылки оставляй как есть, если ссылка уже собрана; экранирование `&` нужно для ВИДИМОГО текста, не для адреса.
- Внутри `<code>`/`<pre>` спецсимволы тоже экранируй (`<` `>` `&`) — иначе Telegram оборвёт блок.

## Антипаттерн: markdown в посте

В теле поста НЕ используй markdown-разметку — она не интерпретируется (`parse_mode=html`).

До (❌) → После (✅):
- `**Всем привет**` → `<b>Всем привет</b>`
- `_важно_` → `<i>важно</i>`
- `~~старое~~` → `<s>старое</s>`
- `` `cmd` `` → `<code>cmd</code>`
- `[текст](url)` → `<a href="url">текст</a>`

Заголовки `#`, списки `- `/`1.`, линии `---` в Telegram не работают как разметка — оформляй абзацами, эмодзи-маркерами «•», переносами строк.

## Таблицы

Telegram таблиц не рендерит. В посте — список «• Поле: значение · Поле: значение». В чат можно написать markdown-таблицу — конвертер сам превратит её в такой список (`| A | B |` → `• A: x · B: y`), но проще сразу писать списком. (Внутренние файлы памяти `content-plan.md`/`insights/*.md` — таблицы там допустимы, они не публикуются.)

## Чек-лист перед постингом

1. В теле черновика только HTML-теги из шпаргалки — ни одной markdown-звёздочки/подчёркивания.
2. Все `<`, `>`, `&` в обычном тексте экранированы.
3. Теги парные и корректно вложены (`</b>` закрывает `<b>`).
4. Ссылки — `<a href="…">…</a>`, не `[…](…)`.
5. Только после этого — `post … --now` или `schedule … --at "…"`.
